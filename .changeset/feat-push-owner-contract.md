vike-push: own push subscriptions by any subject, not just the auth user (#282 / #250) — the push consumer of the shared `@vike-data/kit` `resolveOwner` contract storage (#285) and notifications (#283) adopted. Set `pushOwner: { table: 'organizations', column: 'organization_id' }` (build) plus `VIKE_PUSH_OWNER_COLUMN` + `VIKE_PUSH_OWNER_FROM` (runtime) and a subscription is owned by the subscriber's organization (resolved from the signed-in user's `current_organization_id`): `/push/subscribe` registers every member's device under the org, `sendPush(orgId, …)` fans out to all of them, `saveSubscription`/`removeSubscription`/`sendPush` write/scope by the owner column, and a signed-in user with no org gets `403 no-owner`. The `push_subscriptions` table is now a computed schema contribution (`pushSubscriptionsSchemas`, the vike-stripe `subscriptionSchemas` shape) parameterized by `pushOwner`, declared in `meta` so it plumbs from app config. Unset = the single-user `user_id` -> subject default, byte-for-byte. vike-push stays decoupled from vike-teams — the app names the table/column/source field. (vike-push has no named-guard axis; it resolves against the default subject.)
